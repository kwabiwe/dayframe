# Tags end-to-end implementation

Date: 2026-07-18

## Product decision

- `time_entries` remain the canonical tracked record. No parallel task model was introduced.
- Categories remain the primary classification. Tags are optional workspace-scoped secondary context.
- Persisted `tags` and `time_entry_tags` associations are the read source of truth. Editors hydrate any missing canonical inline tokens from those associations so historical rows do not depend on reparsing descriptions.
- V1 display names accept ASCII letters, numbers, spaces, hyphens, and underscores, with a maximum of 48 characters. Identity is a lowercase token slug; whitespace and hyphens normalize to `-`, underscores remain `_`, and uniqueness is case-insensitive within a workspace.
- A tag typed or selected in an editor is not created until the enclosing entry save commits. Cancelling therefore cannot leave an unused tag behind.

## Design evidence

Figma file `GnSyBZldqvgvHRQ70KWDIL` was inspected at nodes `16:3`, `16:5`, `16:7`, `16:9`, `16:11`, `13:20`, and `14:22`.

- Edit Timer uses a description field with `Type # to add a tag`, an anchored `TAGS` result panel, text-led existing/highlight/create rows, and applied inline metadata.
- Today and Calendar use one small outline tag icon with plain secondary text such as `Planning · Deep work`.
- Tag metadata is never a filled chip, outlined pill, or strongly coloured classification surface.

## Motion contract

- Trigger: entering `#` at a valid token boundary, changing its active query, moving/backspacing out of the token, or selecting an existing/create result.
- Owner: the anchored autocomplete panel. The description input, caret, keyboard, and surrounding form geometry do not animate or reflow.
- Entrance/exit: 140ms ease-out opacity with at most 4pt upward travel. The panel is an overlay beneath the input.
- Selection update: 120ms result highlight colour fade.
- Interruption: rapid typing cancels/replaces the current panel animation; result calculation follows the current caret/query; repeated selection is idempotent at persistence through the association primary key.
- Async rollback: mobile stores the complete pre-edit bootstrap snapshot, applies description and tag metadata optimistically, and restores that snapshot if persistence fails. Queued timer starts retain the desired `tagNames` payload for later event-first reconciliation.
- Reduce Motion: panel travel is removed; state changes are immediate or opacity-only and the highlight update is immediate.
- Accessibility: result rows are at least 44pt/px; existing results and the create action have distinct VoiceOver/ARIA labels; the native Calendar accessibility label includes serialized tag text.

## Data and ownership boundary

- Local schema: `packages/db/migrations/001_init.sql` plus the idempotent existing-database upgrade in `packages/db/migrations/002_tags.sql`. The setup runner now applies every ordered local migration before seeding.
- Hosted migration: `supabase/migrations/202607180001_tags.sql`.
- Hosted state: migration application was confirmed by the user on 2026-07-19. Production UI/API smoke testing remains separate release evidence.
- The join stores `workspace_id` and uses composite foreign keys to both the entry and tag workspace identities. Supabase policies verify both sides of an association.
- Timer starts still pass through `activity_events`; tag associations are written in the same event transaction after the derived entry exists.
- Description updates and their complete desired association set share one database transaction.
- Swift receives `tagText` in native Calendar presentation model version 2 and only renders it. React remains responsible for authenticated bootstrap data, mutations, routing, editors, and offline queue state.
