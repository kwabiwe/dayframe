# Tags end-to-end implementation

Date: 2026-07-18

## Product decision

### 2026-07-19 editor refinement and superseded contract

The original PR #82 regression contract conflicted with the accepted follow-up direction: `docs/dayframe-regression-checklist.md` said selecting a tag must retain a canonical `#token` in Description and that removing the token detaches the tag. The follow-up product decision supersedes that token-as-storage editing model. Inline `#` text is now temporary input only; selecting or creating a tag removes the active token from Description and adds the tag to separate editor state. Existing associations hydrate that selected-tag state without injecting tokens into persisted descriptions. The PR description must call out this deliberate contract replacement.

- Mobile keeps manual `#` entry and adds a compact, borderless `Add a tag` shortcut beneath Description. The shortcut focuses Description and inserts `#` at the caret with sensible spacing.
- Web adds a tag-icon action beside Description. It opens an anchored search/select/create popover without inserting `#`; manual `#` entry remains available.
- Selected tags render through the existing quiet outline-icon metadata treatment, not category-like pills. Removing a selected tag acts on editor state, never by rewriting Description.
- Saves continue to submit one clean Description plus the complete desired tag-name set transactionally. Cancel and failure rollback restore both fields together; offline queued starts/edits preserve desired tag names.

Refinement motion contract:

- Trigger: mobile `Add a tag`, manual `#`, web tag-icon activation, typing/filtering, selection/create, outside/Escape dismissal, or removal of a selected tag.
- Owner: the existing React Native editor owns the mobile autocomplete; the web tag control owns its anchored popover. Inputs and surrounding form layout remain stable.
- Entrance/update/exit: reuse the existing restrained autocomplete opacity/small-travel timing. Web uses the nearest existing raised popover treatment; mobile uses the existing 140 ms tag panel path. Selected metadata updates in place without category-chip motion.
- Surrounding layout: the overlay does not push Description, Category, or Start time. The metadata row may wrap naturally at Dynamic Type/phone widths without an animated global reflow.
- Interruption: rapid typing replaces the active query deterministically; repeated selection is idempotent; outside press/Escape closes only the current popover; stale focus callbacks cannot reopen it.
- Async outcome: selection stays draft-only until the enclosing save. Optimistic save failure restores the exact prior Description and selected tags. No Undo/timeout applies.
- Accessibility: Reduce Motion removes travel, focus moves to the intended search/Description input, VoiceOver/ARIA distinguishes existing/create/remove actions, and every action retains a 44 pt/px hit target even when its visible mobile button is visually compact.

- `time_entries` remain the canonical tracked record. No parallel task model was introduced.
- Categories remain the primary classification. Tags are optional workspace-scoped secondary context.
- Persisted `tags` and `time_entry_tags` associations are the read source of truth. Editors hydrate separate selected-tag state from those associations; they do not inject canonical tokens into Description.
- V1 display names accept ASCII letters, numbers, spaces, hyphens, and underscores, with a maximum of 48 characters. Identity is a lowercase token slug; whitespace and hyphens normalize to `-`, underscores remain `_`, and uniqueness is case-insensitive within a workspace.
- A tag typed or selected in an editor is not created until the enclosing entry save commits. Cancelling therefore cannot leave an unused tag behind.

## Motion contract

- Trigger: entering `#` at a valid token boundary, changing its active query, moving/backspacing out of the token, or selecting an existing/create result.
- Owner: the anchored autocomplete panel. The description input, caret, keyboard, and surrounding form geometry do not animate or reflow.
- Entrance/exit: 140ms ease-out opacity with at most 4pt upward travel. The panel is an overlay beneath the input.
- Selection update: 120ms result highlight colour fade.
- Interruption: rapid typing cancels/replaces the current panel animation; result calculation follows the current caret/query; repeated selection is idempotent at persistence through the association primary key.
- Async rollback: mobile stores the complete pre-edit bootstrap snapshot, applies description and tag metadata optimistically, and restores that snapshot if persistence fails. Queued timer starts retain the desired `tagNames` payload for later event-first reconciliation.
- Reduce Motion: panel travel is removed; state changes are immediate or opacity-only and the highlight update is immediate.
- Accessibility: result rows are at least 44pt/px; existing results and the create action have distinct VoiceOver/ARIA labels; the native Calendar accessibility label includes serialized tag text.

## Data and ownership boundary

- Local schema: `packages/db/migrations/001_init.sql` plus the idempotent existing-database upgrade in `packages/db/migrations/002_tags.sql`. The setup runner now applies every ordered local migration before seeding.
- Hosted migration: `supabase/migrations/202607180001_tags.sql`.
- Hosted state: migration application was confirmed by the user on 2026-07-19. Production UI/API smoke testing remains separate release evidence.
- The join stores `workspace_id` and uses composite foreign keys to both the entry and tag workspace identities. Supabase policies verify both sides of an association.
- Timer starts still pass through `activity_events`; tag associations are written in the same event transaction after the derived entry exists.
- Description updates and their complete desired association set share one database transaction.
- Swift receives `tagText` in native Calendar presentation model version 2 and only renders it. React remains responsible for authenticated bootstrap data, mutations, routing, editors, and offline queue state.

## 2026-07-19 physical-device polish follow-up

TestFlight build 56 physical-iPhone evidence showed the dark autocomplete surface blending into the edit card. The follow-up deliberately raises contrast by using the stronger light surface in dark mode and a darker inset surface in light mode. It also replaces the earlier intricate outline icon with the simpler Toggl-referenced solid rounded tag silhouette and punched hole across mobile and web React metadata/actions.

Selected mobile editor tags are now individually tappable removal actions. A tap changes only `selectedTagNames` in the current editor draft; the enclosing checkmark save remains the sole persistence boundary. Dismissing the sheet rehydrates the original persisted associations, so an accidental tap followed by closing the card cannot detach a tag. VoiceOver announces each action as `Remove tag <name>` and explains that saving confirms removal.
